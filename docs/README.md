## ü§ù Contributing to Documentation

### Documentation Standards

1. **Markdown Format:** All documentation uses GitHub-flavored markdown
2. **Clear Structure:** Use headers, lists, and code blocks appropriately
3. **Examples:** Include practical examples where applicable
4. **Links:** Link to related documentation
5. **Date Stamps:** Include last updated date

### Adding New Documentation

1. **Determine Category:** Place in appropriate directory (user/, developer/, etc.)
2. **Follow Template:** Use existing docs as templates
3. **Update Index:** Add to this README.md
4. **Cross-Link:** Link from related documentation
5. **Review:** Have documentation reviewed before committing

### Archiving Old Documentation

1. **Move to Archive:** Place in docs/archive/ with appropriate subdirectory
2. **Update Links:** Update any links pointing to archived docs
3. **Add Note:** Add archival date and reason to document
4. **Update Index:** Remove from active listings
